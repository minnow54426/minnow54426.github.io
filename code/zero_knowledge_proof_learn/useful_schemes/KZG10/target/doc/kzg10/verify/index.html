Proof verification using bilinear pairings

This module implements the verification algorithm for KZG10 proofs.

§Mathematical Background

Given a commitment C, evaluation point z, claimed value v, and proof π, verification checks that the proof is valid.

§The Pairing Equation

The verifier checks the following pairing equation:

e(π·H, s·H) ?= e(C - v·G, H) · e(π, H)

Where:

  • e(·,·) is the bilinear pairing operation: G1 × G2 → GT
  • π is the proof/witness (G1 element)
  • C is the commitment (G1 element)
  • v is the claimed value (scalar)
  • G, H are generators of G1, G2
  • s is the secret (embedded in SRS)

§Why This Works

If the proof is valid (π = Q(s)·G where Q = (P-v)/(x-z)):

LHS = e(Q(s)·G·H, s·H)
    = e(Q(s)·G, H)^(s)
    = Q(s)·e(G, H)^s

RHS = e((P(s)-v)·G, H) · e(Q(s)·G, H)
    = (P(s)-v)·e(G, H) · Q(s)·e(G, H)
    = Q(s)·s·e(G, H)           [since P(s)-v = Q(s)·(s-z)]
    = Q(s)·e(G, H)^s

The two sides are equal when the proof is valid.

§Security

  • Soundness: Invalid proof will fail with overwhelming probability
  • Knowledge soundness: Prover must know Q(x) to generate valid π
  • Constant-time: Verification time is O(1), independent of degree!
