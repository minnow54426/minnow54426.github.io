Proof and commitment data structures

This module defines the core data structures used in KZG10 polynomial commitments:

  • EvaluationProof: A proof that a polynomial evaluates to a claimed value
  • Opening: A complete opening containing point, value, and proof

Note: The Commitment type is defined in the commit module.

§Mathematical Background

In KZG10, we commit to a polynomial P(x) of degree d by computing:

C = P(s) · G

where s is the structured reference string (SRS) secret, and G is the generator of G1.

To prove that P(z) = v, we compute the quotient polynomial:

Q(x) = (P(x) - v) / (x - z)

and the evaluation proof:

π = Q(s) · G

Verification uses the pairing equation:

e(π·H, s·H) = e(C-v·G, H)·e(π, H)

which simplifies to checking that the quotient was computed correctly.

§Security Properties

  • Binding: Given commitment C = P(s)·G, computationally infeasible to find Q ≠ P with Q(s)·G = C under the discrete log assumption in G1.

  • Hiding: Commitment C = P(s)·G reveals nothing about polynomial P under the decisional Diffie-Hellman assumption in G1.

  • Zero-knowledge: The proof π = Q(s)·G reveals only that P(z) = v, nothing else about P, under the q-PBDH assumption.

§Size Characteristics

  • Commitment: 48 bytes (compressed G1 point)
  • EvaluationProof: 48 bytes (compressed G1 point)
  • Opening: 48 + 48 + 32 = 128 bytes (proof + point + field element)
