Comprehensive error type for all KZG10 operations

Each error variant includes context to help diagnose issues without revealing cryptographic secrets.

ยงSecurity Note

None of these error variants leak information about:

  • The secret s used in trusted setup

  • Private keys or other sensitive data

DegreeTooLarge

Polynomial degree exceeds SRS maximum capacity

This occurs when trying to commit to a polynomial with degree higher than what the SRS was initialized for.

ยงWhen This Occurs

  • During commitment: If poly.degree() > srs.max_degree()
  • During proof generation: If evaluation point requires higher degree
  • During batch operations: If accumulated degree exceeds capacity

ยงSecurity Implications

This error reveals the requested and maximum degrees, which is safe because these are public parameters. No secret information is leaked.

ยงExample

    // SRS initialized for max degree 100

    Err(err)

IoError(Error)

I/O error during file operations

Propagates std::io::Error for operations like:

  • Loading SRS from disk
  • Saving proof batches
  • Reading configuration files

ยงWhen This Occurs

  • File not found when loading SRS
  • Permission denied when writing proofs
  • Disk full when saving large batches
  • Network errors when loading from remote storage

ยงSecurity Implications

I/O errors can leak file paths and other system information, but this is acceptable because:

  1. File paths are local to the system
  2. No cryptographic secrets are revealed
  3. This information is needed for debugging

ยงWhy We Use From Trait

The #[from] attribute on std::io::Error allows automatic conversion from io::Error to our Error type, making error handling with the ? operator seamless:

use std::fs::File;
use kzg10::Result;

fn load_file<T>(path: &str) -> Result<T> {
    // This automatically converts io::Error to Error::IoError
    let _file = File::open(path)?;
    // ... parse file into T
    unimplemented!()
}
