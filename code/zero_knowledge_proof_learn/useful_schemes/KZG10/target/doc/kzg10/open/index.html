Proof generation (opening)

This module implements the “opening” operation which generates a proof that a polynomial evaluates to a specific value at a point.

§Mathematical Background

To prove P(z) = v for polynomial P, point z, and claimed value v:

  1. Compute the quotient polynomial:

    Q(x) = (P(x) - v) / (x - z)

  2. The proof is the commitment to Q(x):

    π = Q(s)·G

§Why This Works: The Fundamental Theorem of Algebra

The key insight is the Factor Theorem from polynomial algebra:

(x - z) divides (P(x) - v)  ⟺  P(z) = v

§Proof of the Factor Theorem

Forward direction (⇒): If (x - z) divides (P(x) - v), then there exists Q(x) such that:

P(x) - v = (x - z) · Q(x)

Evaluating at x = z:

P(z) - v = (z - z) · Q(z) = 0 · Q(z) = 0

Therefore: P(z) = v

Backward direction (⇐): If P(z) = v, then (P(x) - v) has a root at z:

P(z) - v = 0

By the Fundamental Theorem of Algebra, any polynomial with root z can be factored as:

P(x) - v = (x - z) · Q(x)

for some quotient polynomial Q(x) of degree deg(P) - 1.

§Cryptographic Significance

This algebraic fact gives us a cryptographic proof system:

  1. Correctness: The quotient Q(x) exists iff P(z) = v
  2. Soundness: If P(z) ≠ v, division by (x - z) fails (remainder ≠ 0)
  3. Binding: Commitment to Q(x) proves knowledge of valid quotient

The verifier checks that the prover knows a Q(x) satisfying:

P(x) - v = (x - z) · Q(x)

Without knowing P or Q directly, the verifier uses pairings to check that this relationship holds “in the exponent” with the secret s.

§Verification

The verifier checks the pairing equation:

e(π·H, s·H) ?= e(C - v·G, H) · e(π, H)

If this holds, then with overwhelming probability P(z) = v.

§Algorithm

The opening algorithm performs the following steps:

  1. Evaluate: Compute v = P(z) using polynomial evaluation (Horner’s method)
  2. Subtract: Compute polynomial R(x) = P(x) - v
  3. Divide: Compute Q(x) = R(x) / (x - z) using synthetic division
  4. Commit: Compute π = Q(s)·G using multi-scalar multiplication with SRS

§Performance Characteristics

For a polynomial of degree n:

  • Evaluation: O(n) using Horner’s method
  • Synthetic Division: O(n) - single pass through coefficients
  • Commitment to Quotient: O(n) using optimized MSM (or O(n/log n) with Pippenger)

Total time: O(n) where n is the polynomial degree

Example timings on modern hardware (AMD Ryzen 9 5900X):

  • Degree 2¹² (4096): ~15ms total
  • Degree 2¹⁶ (65536): ~80ms total
  • Degree 2²⁰ (1M): ~500ms total

§Security Properties

§Completeness

If P(z) = v and the prover follows the algorithm honestly, the verification will always accept the proof. This is guaranteed by the algebraic correctness of the quotient polynomial computation.

§Soundness

Under the q-SBDH (Strong Bounded Diffie-Hellman) assumption, a malicious prover cannot create a valid proof π for a false statement (P(z) ≠ v).

The soundness error is approximately 1/|Fr| ≈ 2^(-254) for BLS12-381, which is cryptographically negligible.

§Zero-Knowledge

Under the q-PBDH (Pairing-Based Diffie-Hellman) assumption, the proof π reveals only that P(z) = v, and nothing else about the polynomial P.

Specifically, a simulator can produce indistinguishable proofs without knowing P, only given (z, v, srs).

§Synthetic Division: The Key Algorithm

Synthetic division is a specialized algorithm for dividing a polynomial by a linear term (x - z). It’s much faster than general polynomial division.

§Algorithm Derivation

Given polynomial P(x) = cₙxⁿ + cₙ₋₁xⁿ⁻¹ + … + c₁x + c₀, we want to divide by (x - z) to get quotient Q(x) = qₙ₋₁xⁿ⁻¹ + qₙ₋₂xⁿ⁻² + … + q₀.

Standard polynomial long division gives us the recurrence:

qₙ₋₁ = cₙ
qₙ₋₂ = cₙ₋₁ + z·qₙ₋₁
qₙ₋₃ = cₙ₋₂ + z·qₙ₋₂
...
q₀ = c₁ + z·q₁
remainder = c₀ + z·q₀

§Why This Works

Each step of the recurrence eliminates the highest-degree term:

Start: P(x) = cₙxⁿ + cₙ₋₁xⁿ⁻¹ + … + c₀

After setting qₙ₋₁ = cₙ:

P(x) - qₙ₋₁·xⁿ⁻¹·(x - z)
= cₙxⁿ + cₙ₋₁xⁿ⁻¹ + ... - cₙxⁿ + cₙ·z·xⁿ⁻¹ - ...
= (cₙ₋₁ + cₙ·z)xⁿ⁻¹ + ...

The coefficient of xⁿ⁻¹ becomes (cₙ₋₁ + z·qₙ₋₁), which is exactly qₙ₋₂.

This pattern continues, eliminating one degree at each step.

§Computational Complexity

  • Time: O(n) - single pass through coefficients
  • Space: O(n) - store quotient coefficients
  • Operations: n multiplications + n additions

Compared to general polynomial division which is O(n²), synthetic division is dramatically faster because it exploits the special structure of the divisor.
